## 2. الملخص والمشكلة (Abstract & Problem Statement)

### المقدمة (4–10 جمل)
التجارة الإلكترونية في ليبيا تنمو بسرعة، لكن قطاع العطور بقي خارج هذا التحول الرقمي. شراء العطر عبر الإنترنت يصطدم بعقبة جوهرية: العميل لا يستطيع شمّ المنتج قبل الشراء، فيتردد ويتراجع عن إتمام الطلب. كما تفتقر السوق الليبية إلى منصات عربية أولاً (Arabic-first) تدمج الذكاء الاصطناعي في تجربة التسوق، وتربط الطلب مباشرة بشركات التوصيل المحلية. مشروع "شمّة" (shama.ly) يعالج هذه الفجوة عبر منصة ويب ثنائية اللغة (عربي/إنجليزي) تستخدم نماذج ذكاء اصطناعي متعددة لمساعدة العميل على اختيار العطر المناسب كأنه يتحدث مع بائع خبير، مع إدارة كاملة للمتجر عبر لوحة تحكم وبوت تيليجرام.

### المشكلة (The Problem)
- العميل لا يستطيع تجربة (شمّ) العطر قبل الشراء أونلاين → تردد وانخفاض في إتمام الطلبات.
- صعوبة اختيار العطر المناسب من بين عشرات المنتجات دون استشارة خبير.
- غياب منصات ليبية تدعم العربية بالكامل (RTL) مع تجربة تسوق ذكية.
- إدارة المتجر يدوياً (الطلبات، المراجعات، التوصيل، التسويق) تستهلك وقت صاحب المتجر.

## 3. الحل والأهداف (The Solution & Objectives)

### الحل المقترح
منصة ويب متكاملة (**shama.ly**) تجمع:
1. **مساعد ذكاء اصطناعي** — شات بوت يعرف كامل الكتالوج، اختبار شخصية عطرية (Quiz) يرشّح أفضل 3 عطور، وبحث ذكي بالوصف الطبيعي.
2. **حل مشكلة "الشمّ"** — نظام عينات (Samples 3–30ml) يتيح تجربة العطر بسعر رمزي قبل شراء الزجاجة الكاملة.
3. **تجارة إلكترونية كاملة** — سلة، كوبونات خصم، أطقم هدايا، قائمة أمنيات، تتبع الطلب لحظياً عبر تكامل مباشر مع شركة التوصيل **Vanex**.
4. **إدارة ذكية** — لوحة تحكم إدارية + بوت تيليجرام لإدارة المتجر من الهاتف، وفلترة المراجعات تلقائياً بالذكاء الاصطناعي.
5. **تسويق آلي** — تحليل ليلي لاهتمامات كل مستخدم وإرسال إيميلات مخصصة عند وصول عطور تناسب ذوقه.

### الأهداف (Objectives)
- رفع ثقة العميل في شراء العطور أونلاين عبر العينات والمراجعات الموثوقة.
- تقديم توصيات شخصية دقيقة باستخدام نماذج LLM حديثة.
- دعم كامل للغة العربية (RTL) والإنجليزية في كل شاشة.
- أتمتة إدارة المتجر والتسويق لتقليل الجهد اليدوي.
- بناء بنية قابلة للتوسع (Scalable) بتكلفة تشغيل منخفضة (استضافة ثابتة + Serverless).

## 4. المنهجية وأدوات النظام (Methodology & System Architecture)

### مخطط معمارية النظام (System Architecture Diagram)
*(ارسمه في البوستر كصناديق وأسهم)*

```
المستخدم (متصفح / هاتف)
        │
        ▼
الواجهة الأمامية — React 18 + TypeScript + Vite + Tailwind (shama.ly)
        │
        ▼
Supabase (Backend as a Service)
 ├── PostgreSQL  (المنتجات، الطلبات، المراجعات، الكوبونات)
 ├── Auth        (تسجيل دخول + OTP + Google OAuth)
 ├── Storage     (صور المنتجات)
 └── Edge Functions (Deno) — 7 وظائف:
       تحليل ملف المستخدم، مطابقة المنتجات الجديدة،
       المنسّق الليلي، إرسال الإيميلات، إلغاء الاشتراك، مزامنة Vanex
        │                          │
        ▼                          ▼
OpenRouter AI Gateway        Vanex Delivery API
 ├── GPT-5.2 (شات بوت/توصيات)   (إنشاء وتتبع الشحنات)
 ├── Claude Sonnet 4.6 (فلترة المراجعات)
 └── Gemini Image (توليد صور المنتجات)

بوت تيليجرام (Node.js + Telegraf) ──► نفس قاعدة بيانات Supabase
```

### التقنيات المستخدمة (Tech Stack)
**Frontend:** React 18, TypeScript, Vite, Tailwind CSS, shadcn/ui (Radix), Framer Motion
**Backend:** Supabase (PostgreSQL, Auth, Storage, Edge Functions/Deno)
**AI:** OpenRouter — GPT-5.2, Claude Sonnet 4.6, Gemini Image
**التكاملات:** Vanex API (توصيل), Telegram Bot (Telegraf)
**الفيديو التسويقي:** Remotion (فيديو مولّد برمجياً بنسختين عربي/إنجليزي)
**الاختبارات:** Playwright (E2E)
**الاستضافة:** shama.ly — Libyan Spider (cPanel)

### المنهجية
تطوير تكراري (Iterative/Agile): تحليل المتطلبات → تصميم → تنفيذ → اختبار E2E آلي → نشر، مع تحسين مستمر للأداء وتجربة المستخدم.

## 5. النتائج والمميزات (Results & Features)

### لقطات شاشة (Screenshots) — ضع 4–6 صور

- **3** نماذج ذكاء اصطناعي متخصصة (محادثة، فلترة، توليد صور)
- **7** وظائف خلفية Serverless (Edge Functions)
- **8** حِزم اختبارات E2E آلية (Playwright)
