# وثيقة المتطلبات الوظيفية - متجر شمّة للعطور الفاخرة

**المشروع:** منصة شمّة للتجارة الإلكترونية للعطور الفاخرة
**الإصدار:** 2.0
**التاريخ:** 2026-03-26
**اللغة:** عربي (المصطلحات التقنية بالإنجليزية)

---

## 1. نظرة عامة

منصة شمّة هي متجر إلكتروني متكامل لبيع العطور الفاخرة في ليبيا، مبني باستخدام React/TypeScript/Supabase مع تكامل الذكاء الاصطناعي (Gemini AI). يشمل النظام بوابة دفع Moamalat وخدمة توصيل Vanex.

## 5. دورة حياة الطلب (Order Lifecycle)

### 5.1 مسار الطلب الكامل (محدّث)

```
┌─────────────────────────────────────────────────────────┐
│ العميل يضع الطلب                                        │
│ status: "pending"                                        │
│ يختار طريقة الدفع: COD أو Moamalat                       │
├─────────────────────────────────────────────────────────┤
│                                                          │
│ ┌──── إذا Moamalat ────┐  ┌──── إذا COD ────────┐      │
│ │ فتح LightBox         │  │ payment_status:      │      │
│ │ → نجاح: paid          │  │ "cod_pending"        │      │
│ │ → فشل: failed         │  └────────────────────┘      │
│ │ → إلغاء: يعود للصفحة │                                │
│ └──────────────────────┘                                │
├─────────────────────────────────────────────────────────┤
│ المسؤول يؤكد الطلب                                       │
│ status: "confirmed"                                      │
│ → تلقائي: إنشاء شحنة في Vanex API                       │
├─────────────────────────────────────────────────────────┤
│ Vanex يعين رقم التتبع                                    │
│ status: "processing"                                     │
│ vanex_tracking_id: محفوظ                                 │
├─────────────────────────────────────────────────────────┤
│ Vanex يشحن الطلب                                         │
│ status: "shipped"                                        │
├─────────────────────────────────────────────────────────┤
│ Vanex يوصل الطلب                                         │
│ status: "delivered"                                      │
│ إذا COD: payment_status → "cod_collected"                │
└─────────────────────────────────────────────────────────┘
```

### 5.2 حالات الطلب

| الحالة | الوصف | اللون |
|--------|-------|-------|
